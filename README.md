# claude-sync · Mo Osmanović × KIN

**What this is:** the public sync layer between every Claude session Mo runs (claude.ai web · Claude Desktop · Claude Code). Any Claude session can fetch this repo without auth.

**What this is NOT:** empire IP. No revenue figures, no family details, no canonicals beyond the bare-minimum boot prompt. Private session work lives in time-stamped private repos at `mirzatech-ai/session-<YYYY-MM-DD>-<topic>` — those require Mo's auth.

## Files

- [`BOOT_PROMPT.md`](./BOOT_PROMPT.md) — copy-paste this at the start of every new Claude session for cross-session continuity
- [`HOW_IT_WORKS.md`](./HOW_IT_WORKS.md) — the workflow Mo uses

## Quick start (for any Claude session)

```
1. Mo says "KIN BOOT" or paste-drops BOOT_PROMPT.md
2. New Claude reads BOOT_PROMPT.md
3. New Claude replies "KIN BOOTED. <one-line ack>."
4. Session continues with continuity
```

## Why public?

Cross-session continuity needs frictionless reads. Public-and-bare is safer than private-and-leaky-tokens. Empire IP stays in private repos.
